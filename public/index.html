☁️ Como publicar sua sincronização
Passo a passo pra criar seu próprio servidor gratuito na Vercel. Leva uns 15-20 minutos, sem precisar programar.
Acesse vercel.com/signup e crie uma conta grátis (pode usar login do GitHub, Google ou e-mail).
Acesse github.com/new, crie um repositório (pode ser privado, ex: stakebet-sync) e faça upload dos 3 arquivos abaixo nele (botão "Add file" → "Upload files").
Esses 3 arquivos já estão prontos, só baixar e subir no GitHub:
Importante: o arquivo data.js precisa ficar dentro de uma pasta chamada api no repositório (ex: api/data.js).
No painel da Vercel: Storage → Create Database → Postgres. Dê um nome (ex: stakebet-db) e crie — fica no plano gratuito.
Na Vercel: Add New → Project → selecione o repositório que você criou. Antes de publicar, vá em Environment Variables e adicione uma chamada ACCESS_PIN com uma senha só sua (é o PIN que você vai usar aqui no app). Depois, na aba Storage do projeto, conecte o banco criado no passo 4. Clique em Deploy.
Depois do deploy, a Vercel te dá uma URL (ex: https://stakebet-sync-xxx.vercel.app). Cola ela no campo "URL do servidor" acima, junto com o PIN que você criou, clica em "Salvar conexão" e depois "Enviar". Repete a conexão (URL + PIN) no outro aparelho e clica em "Baixar".
Dúvidas comuns: erro 401 = PIN diferente do configurado na Vercel. Erro de CORS/fetch = confira se copiou a URL completa, sem barra no final.
